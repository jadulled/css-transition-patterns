CSS Transition patterns

Discard

Based on the Saving indicator pattern from UI Transitions (look into the orientation transition), but in reverse mode, make a discard effect to a canvas

Description

Clicking the card close button will trigger an animation of discard to the right

Working Demo

Working demo

The HTML needed

We are using the animation tools from WestCiv that allows to change the animation properties in real-time with graphical interface. Making the task a lot easier. With the Daneden html code we trigger the animations (by jquery changes the button style when clicked)

  1. Head Section:

    <link rel="stylesheet" href="stylesheets/style.css">
<script src="http://ajax.googleapis.com/ajax/libs/jquery/1.6.4/jquery.min.js"></script>

  2. Body:

        <div id="container">
      <div id="card">
           <p>Test Card.</p>
           <a class="button" href="#" onclick="$('#card').addClass('discardOnClose');">Close</a>
      </div>
    </div>

    Adding the class "discard" to the event "onclick" with jquery we can trigger the action.

    <a class="button" href="#" onclick="$('#card').addClass('discardOnClose');">Close</a>

The CSS needed

In css are two things to define:

  1. Give a name to the animation and use the global properties like animation name, duration, delay, iteration count, type of easing, and direction

    Let's see:

    
.discardOnClose{
/*W3C*/
  animation-name: discardOnClose;
  animation-duration: 1s;
  animation-delay: 0s;
  animation-iteration-count: 1;
  animation-timing-function: ease-in-out;
  animation-direction: normal;
  animation-fill-mode:forwards;

/*For webkit*/
  -webkit-animation-name: discardOnClose;
  -webkit-animation-duration: 1s;
  -webkit-animation-delay: 0s;
  -webkit-animation-iteration-count: 1;
  -webkit-animation-timing-function: ease-in-out;
  -webkit-animation-direction: normal;
  -webkit-animation-fill-mode:forwards;

/*For Mozilla*/
  -moz-animation-name: discardOnClose;
  -moz-animation-duration: 1s;
  -moz-animation-delay: 0s;
  -moz-animation-iteration-count: 1;
  -moz-animation-timing-function: ease-in-out;
  -moz-animation-direction: normal;
  -moz-animation-fill-mode:forwards;
}

  2. The property animation-fill-mode specify whether the effects of an animation are visible before or after it plays. A value of backwards causes the styles in the first keyframe to be applied before the animation runs. A value of forwards causes the styles in the last keyframe to be applied after the animation runs. A value of both does both.

    Then, according to browser define the keyframes: They are instances where the properties will change over the animation.

    @keyframes discardOnClose {
  0% { 
        transform: scale(0.6); 
        transform-origin: 47% 53%; /*setting the initial values for the properties that will change*/
  }
  25% {
        transform: scale(1.4) 
        rotate(-56deg);
        transform-origin: 37% 53%; /*changing the transformation origin makes the center displacement*/
  }
  100% {transform: scale(1.0) 
        rotate(344deg) 
        transform-origin: 50% 50%; 
        translateX(600px) translateY(800px); /*move to left */
  }
}

@-webkit-keyframes discardOnClose {
  0% {
-webkit-transform: scale(0.6);
-webkit-transform-origin: 47% 53%;
  }
  25% {
-webkit-transform: scale(1.4) rotate(-56deg);

  }

-webkit-transform-origin: 50% 50%;
  }
}
